# Internal Docs

This folder contains historical notes, investigation reports, and implementation plans that were useful during recovery and stabilization.

These files are intentionally separated from the public-facing docs in `docs/` to keep the repository landing experience clean and to avoid confusing day-to-day operators.

Notes:

- Many documents in `docs/internal/legacy/` retain legacy “MacAgent” naming and older port examples.
- StarAgent’s current public docs are in `docs/` (Quickstart, WebUI, CLI, MCP, evaluation, compatibility).
- Internal docs may include outdated assumptions. Treat them as reference material, not the runbook.
